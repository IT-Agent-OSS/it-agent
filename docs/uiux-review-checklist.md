# UIUX Review Checklist

## Purpose

`uiux` ロールが毎回同じ観点で見られるようにするための標準チェックです。

## First Impression

- 最初の3秒で何のツールか分かるか
- 最初に触る場所が明確か
- 見た目が弱くて価値が伝わっていないか

## Hierarchy

- 重要な情報が上に来ているか
- 主ボタンと補助ボタンの差が明確か
- どこを見るべきか迷わないか
- タイトルや導入文が大きすぎて、主機能の面積を圧迫していないか

## Flow

- 入力から結果までの流れが自然か
- 途中で詰まりそうな場所がないか
- エラー時に次に何をすればよいか分かるか
- 主操作から主要結果まで、できれば一画面で追えるか
- 一画面で無理がある場合、画面切り替えやステップ遷移で理解しやすく分けられているか

## Interaction

- 押せるものが押せるように見えるか
- 反応が分かりにくい操作がないか
- ローディングや待ち時間が不安を生まないか

## Visual Quality

- 余白が詰まりすぎていないか
- 余白不足で上下左右が窮屈に見えないか
- 強調の使い方が単調すぎないか
- 見た目がテンプレ感だけで終わっていないか
- 横幅が広すぎて、標準的な動画フレームで右端が切れないか
- 日本語向けデモなのに、説明が英語主体になっていないか
- 過去runの見た目を流用しすぎて、今回の題材に最適化されていない部分がないか

## Demo Readiness

- 動画で見せる主場面が明確か
- 改善前後の差分を見せやすいか
- スクリーンショット1枚でも魅力が伝わるか
- ブラウザツールなら、画像合成よりブラウザ操作録画の方が価値伝達に向くか
- 収録フレームで主機能が見切れず、説明より機能面が主役になっているか
- 見切れ回避のために縮小する前に、レイアウト再設計や画面切り替えで解決しているか

## Standardization Trigger

以下に当てはまる場合は、UI/UX 標準ルール候補として `improver` に渡す:

- 同種の分かりにくさが繰り返し出る
- 毎回入れた方がよいレイアウト原則がある
- 特定言語やUIスタックに依存しない汎用学習がある
- ただし、固定レイアウトや固定演出を標準化対象にしない
