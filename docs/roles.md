# IT Agent Roles

## Purpose

`it-agent` の内部に持たせる初期ロール定義です。

この段階では、まだ実装ではなく役割定義だけを持ちます。

## Default Roles

### Builder

- ローカルで起動できるツールを作る

### Test

- 起動可否と主要動作を確認する

### Review

- 品質、使いやすさ、OSSとしての見え方を確認する

### Improvement

- Test / Review の結果から、次回の作り方を改善する

### UI/UX

- 見た目、使いやすさ、導線、第一印象を改善する
- 動くが弱いUIを強化する
- 動画やスクリーンショットで価値が伝わる見せ方まで責任を持つ
- 導入説明より、主操作と結果が目立つ構図を優先する
- ブラウザ系成果物では、録画フレーム内の見切れや余白不足をレビュー対象に含める
- 固定テンプレートに寄せるのではなく、毎回その成果物に合う情報設計と演出を選ぶ
- 再利用するのは見た目そのものではなく、判断原則とレビュー観点に留める

### Specialist Roles

- 言語特化や用途特化の役割は初期常設しない
- 必要になった時点で条件付き追加とする
- 追加条件は `agent-composition-rules.md` に従う

## Evolution Rule

- 役割名を増やす前に、既存役割の指示と基準を改善する
- `it-agent` は中身のルール、観点、developer instructions も継続的に進化させる
- UI/UX と動画表現も、固定の型を守るのではなく、毎回の改善を通じて進化させる

## Important Rule

外部に配布するのは `it-agent` 単体であり、各ロールを独立プロダクトとして配布しない。
