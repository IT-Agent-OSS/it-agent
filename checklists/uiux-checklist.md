# UIUX Checklist

## Purpose

`uiux` が見る観点を短く運用しやすくまとめたチェックです。

## Checklist

- 最初の3秒で何のツールか分かるか
- タイトルや概要説明より、主機能の操作面と結果面が主役になっているか
- 主操作が明確か
- 見た目が弱くて価値が落ちていないか
- CLI の場合、first-run example と expected output がすぐ見えるか
- Web の場合、what it does と expected result が above the fold にあるか
- Web の場合、主操作と主要結果が同一画面内で追えるか
- 1画面に収まらない場合、縮小で押し込む前に画面分割やステップ切り替えで見せられるか
- 分類や比較ツールでは、decision rule や confidence が visible か
- 入力から結果まで迷わないか
- 動画で見せる見どころがあるか
- 動画化する場合、1280x720 など最終フレーム基準で上下左右の見切れがないか
- 録画用の縮小表示は最後の手段であり、まずUI構成で収める方針になっているか
- ブラウザ成果物では、静止画つなぎより実ブラウザ操作の録画を優先すべきか確認したか
- 日本語向け発信なら、UI と補足説明が日本語で理解できるか
- 標準化できるUI/UX学習があるか
- 過去の見た目テンプレートをなぞっているだけになっていないか
- 再利用してよいのは原則であり、レイアウトや演出は今回のテーマに合わせて再判断しているか
